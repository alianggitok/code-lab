[ 文字超出省略的处理脚本应用 ]

用法说明

uaxpUI.wordsEllipsis(obj,ellipsisSymb,adjustLen);

原理

逐字递减，判断文字高度是否超出父元素高度，如果超出则继续减少文字，直至不再超出。基于jQuery。

参数

obj: 应用对象
ellipsisSymb: 省略符号html
adjustLen: 修正省略字长

必须

需在html文档中引用“auto-ellipsis.js”

需定义对象style的height或max-height值（一般为line-height×行数），且设置overflow为hidden

示例

CSS

.words{
	max-height:48px;
	line-height:16px;
	font-size:12px;
	overflow:hidden
}
.ua-ellipsissymb{color:red}

Javascript

$(function(){
	uaxpUI.wordsEllipsis('.words','<span class="ellipsissymb">…</span>',10);
});

HTML

<p class="words">
	文字……文字
</p>

输出

The omnipresent menu can´t be applied in many cases because of the screen size. The menu probably would have to be in a vertical way and it could be very uncomfortable if for every single page people would need to scroll down all the menu to see the content. So, only use omnipresent menu if your site has only three links or less on its top navigation and, because of that, it can be horizontal. The solution is to put the navigation on the first page and on the others pages use a link at the top to come back to the first page or use breadcrumbs.

因为屏幕大小的限制，原本无所不在的菜单将无法被完全应用。如果为每个页面都直接应用上相同的菜单，访问者就必须做滚动操作，翻掉菜单才能看到内容。这会让人非常不适。所以只有当你的网站在顶部导航中有少量菜单链接时才考虑把它们放置到每个页面中。为此，它还得是横向水平的。 解决方案是，把导航放置到第一个页面，而在其他页面中尽可能的使用一个链接连向第一个页面，或者使用面包屑式的导航。
